= Shared Responsibility
:page-toc: right

Sovereign ist im Cloud-Umfeld immer geteilt:

== Cloud Provider (typisch)
* Physische Sicherheit, Rechenzentrum, Basishardware
* Basis-Availability von Control Plane / Services
* Dokumentation zu Regionen, Subprozessoren, Compliance-Programmen

== Cloud Consumer / Betreiber (WAF++ Fokus)
* Datenklassifikation und Data-Residency-Policy
* Region Pinning, Netzwerk-/Egress-Kontrolle
* IAM, Privileged Access, Break-Glass, Logging
* Schlüsselmanagement (Ownership, Rotation, Deletion)
* Inventar der Abhängigkeiten (Subprozessoren, Drittanbieter, SaaS)
* Incident/Change/Backup/Restore/Exit Prozesse und Evidenz

NOTE: Viele Provider liefern „compliance artefacts“. Für WAF++ zählt: Welche Teile sind *konfigurierbar/erzwingbar* und welche nur *vertraglich*? Beides muss sauber dokumentiert sein.
