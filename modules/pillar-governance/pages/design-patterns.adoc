= Design Patterns
:page-toc: right

== Pattern 1: Region-Pinned Landing Zone
* Dedizierte Accounts/Subscriptions/Projects pro Jurisdiction
* Policy-as-Code: erlaubte Regionen, verbotene Services/Features
* Zentralisiertes Logging in sovereign Storage (Region fix)

== Pattern 2: Sovereign Key Hierarchy
* CMK/BYOK/HYOK je nach Anforderung
* HSM-backed Keys für hochkritische Daten
* Key Rotation & Deletion als Prozess + Nachweis

== Pattern 3: Controlled Egress
* Default-deny Egress, explizite Allow-Lists
* DNS/Proxy/Egress-Gateways mit Logging
* Exfiltration-Detektion (Anomalien)

== Pattern 4: Break-Glass with Strong Guardrails
* Zeitlich begrenzte Admin-Rollen
* MFA/Hardware Keys, Ticketbindung
* Vollständige Protokollierung + regelmäßige Reviews

== Pattern 5: Exit-Ready Architecture
* Datenexport-Mechanismen (inkl. Metadaten)
* Provider-agnostische Formate (z.B. OpenTelemetry, IaC-Standards)
* Regelmäßige Exit-Drills (Testmigration / Restore in Zielumgebung)
