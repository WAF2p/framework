= Region Pinning & Jurisdiction Enforcement
:page-toc: right

== Kontext

Regionale Einschränkungen werden häufig dokumentiert,
aber nicht technisch erzwungen.

Das führt zu:
* Unbeabsichtigten Deployments
* Compliance-Risiken
* Audit Findings

== Zugehörige Controls

* WAF-SOV-010
* WAF-SOV-020
* WAF-SOV-090

== Zielbild

Deployments außerhalb erlaubter Regionen sind technisch unmöglich
oder werden automatisch blockiert.

== Technische Umsetzung

=== Organisationsebene

* Region Allow List
* Deny-Policies für nicht erlaubte Regionen

=== IaC Ebene

* Terraform Variable Validation
* Sentinel / OPA Regeln
* CI Blocker bei Verstoß

=== Monitoring

* Audit Log Alerts bei Region-Verletzung
* Regelmäßige Drift-Scans

== Typische Fehlmuster

* "Nur für Testzwecke" Deployments
* Temporäre Ausnahmen ohne Review
* Shadow Projects ohne Guardrails

== Metriken

* Anzahl blockierter Region-Deployments
* Anzahl Policy-Verstöße pro Monat
* Zeit bis Behebung

== Reifegrad

Level 1 – Region dokumentiert
Level 2 – Teilweise Guardrails
Level 3 – Vollständige Policy Enforcement
Level 4 – Drift Monitoring aktiv
Level 5 – Automatische Remediation
