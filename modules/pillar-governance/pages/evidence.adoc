= Evidenz & Audit
:page-toc: right

Sovereign muss prüfbar sein. Diese Seite beschreibt, welche Evidenztypen im WAF++ üblich sind.

== Evidenztypen
* *Architecture Evidence*: Diagramme, Data Flows, Trust Boundaries
* *IaC Evidence*: Terraform Code, Plan/State, CI Logs, Policy-as-Code
* *Config Evidence*: Exportierte Konfigurationen, Settings, Resource Graph
* *Logging Evidence*: Audit Logs, Admin Actions, Break-Glass, Egress Logs
* *Process Evidence*: Runbooks, Change Records, Review-Protokolle, Drills
* *Contract Evidence*: Subprocessor Listen, DPA, SLA/Compliance Artefakte

== Mindestanforderungen an Evidenz
* Muss datiert und versioniert sein
* Muss eindeutig einer Control-ID zugeordnet werden können (z.B. `WAF-SOV-070`)
* Muss nachvollziehbar sein (Quelle, Verantwortlicher, Scope)

== Audit-Hinweise (MVP)
* Prüfe zuerst Policies (WAF-SOV-010) und Enforcement (WAF-SOV-020)
* Verifiziere dann „Leak Paths“: Backups (WAF-SOV-030), Observability (WAF-SOV-040), Egress (WAF-SOV-090)
* Schließe mit Betriebs-/Exit-Fähigkeit (WAF-SOV-070, WAF-SOV-100)
