= Reifegrad-Modell: Sustainability Maturity
:page-toc: right
:description: WAF++ Sustainability – 5-stufiges Reifegrad-Modell: Blind → Sichtbar → Reported → Optimiert → Carbon-Neutral. Self-Assessment-Checklisten und empfohlener Einstiegspfad.

Das Reifegrad-Modell der Sustainability-Säule bewertet die *organisatorische und technische Maturität*
entlang fünf Stufen — von keiner Messung bis zur aktiven CO₂-Reduktion auf Net-Zero-Pfad.

== Die 5 Reifestufen

[cols="1,2,4"]
|===
|Stufe |Bezeichnung |Beschreibung

|*1*
|*Blind*
|IT-Emissionen unbekannt. Keine Cloud-Carbon-Tools aktiviert. Keine ESG-Reporting-Pflicht bekannt oder adressiert. Sustainability kein Architekturthema.

|*2*
|*Sichtbar*
|Cloud-Carbon-Tools aktiviert. Emissionen werden gelegentlich eingesehen. Keine Targets, keine systematische Attribution nach Workload. Keine Automatisierung.

|*3*
|*Reported*
|Strukturierte, monatliche Emissionsmessung mit Workload-Attribution. Daten fließen in ESG/CSRD-Bericht. Baseline definiert. Erste Optimierungsmaßnahmen (Lifecycle-Policies, Non-Prod Shutdown).

|*4*
|*Optimiert*
|Aktive Reduktionsmaßnahmen operational: Graviton/ARM-Majority, Idle-Elimination, Storage-Lifecycle, Batch Time-Shifting. Reduktionsziele messbar verfolgt. SCI als Metrik etabliert.

|*5*
|*Carbon-Neutral/Positiv*
|Emissionsreduktion auf SBTi 1,5°C-Zielniveau. Verbleibende unvermeidbare Emissionen durch verifizierte Kompensation (Gold Standard Carbon Credits) neutralisiert. Jährlicher Nachweis.

|===

== Self-Assessment: Checkliste Level 2 (Sichtbar)

Eine Organisation ist auf Level 2, wenn alle folgenden Punkte zutreffen:

* [ ] AWS Customer Carbon Footprint Tool, Azure Emissions Impact Dashboard oder GCP Carbon Footprint ist aktiviert
* [ ] Emissionen wurden in den letzten 6 Monaten mindestens einmal manuell exportiert und eingesehen
* [ ] Die Frage "Wieviel CO₂ emittiert unser Cloud-Betrieb?" ist grob beantwortbar (Größenordnung)
* [ ] Wenigstens ein Team nutzt ARM/Graviton-Instanzen oder Lambda arm64
* [ ] CSRD-Pflicht der Organisation ist geklärt (in scope, out of scope, Tochtergesellschaft)

== Self-Assessment: Checkliste Level 3 (Reported)

Eine Organisation ist auf Level 3, wenn alle folgenden Punkte zutreffen:

* [ ] Monatliche Carbon-Footprint-Exporte aus allen genutzten Cloud-Providern eingerichtet
* [ ] Workload-Tagging ermöglicht Emissions-Attribution auf Workload-Ebene
* [ ] Carbon-Daten sind Teil des offiziellen ESG- oder Nachhaltigkeitsberichts (oder Aufbau dafür ist dokumentiert)
* [ ] Alle S3-Buckets haben Lifecycle-Policies
* [ ] Alle CloudWatch Log Groups haben `retention_in_days` gesetzt
* [ ] Non-Production-Umgebungen haben Scheduled-Shutdown-Regeln
* [ ] Sustainability Debt Register existiert und wurde im letzten Quartal aktualisiert

== Self-Assessment: Checkliste Level 4 (Optimiert)

Eine Organisation ist auf Level 4, wenn alle folgenden Punkte zusätzlich zutreffen:

* [ ] Mehr als 60% der EC2/Container-vCPU-Stunden laufen auf ARM/Graviton oder Ampere Altra
* [ ] Mehr als 80% der Lambda-Funktionen nutzen arm64
* [ ] Autoscaling mit Scale-to-Zero für alle zustandslosen Compute-Workloads konfiguriert
* [ ] Alle Batch-Workloads sind auf Off-Peak (22:00–06:00 UTC) oder Flexible-Windows konfiguriert
* [ ] CDN für alle user-facing Anwendungen; HTTP-Kompression auf allen APIs aktiv
* [ ] VPC Endpoints für S3 und DynamoDB (und weitere AWS Services) in allen produktiven VPCs
* [ ] SCI (Software Carbon Intensity) für mindestens die drei wichtigsten Tier-1-Services gemessen
* [ ] Jahres-über-Jahres-CO₂-Reduktionsziel definiert und im letzten Quartalsbericht verfolgt

== Empfohlener Einstiegspfad

[cols="1,2,3,1"]
|===
|Phase |Zeitrahmen |Maßnahmen |Ziel-Level

|*Phase 0: Baseline*
|Woche 1–2
|Carbon-Tools aktivieren, CSRD-Pflicht klären, aktuellen Zustand inventarisieren
|Level 1 → 2

|*Phase 1: Quick Wins*
|Monat 1–2
|S3-Lifecycle-Policies, Log-Retention, Non-Prod Scheduled Shutdown, Lambda auf arm64
|Level 2 → 3

|*Phase 2: Systematisierung*
|Quartal 1–2
|Sustainability Debt Register, CDN + Kompression, VPC Endpoints, Quarterly Reviews, ESG-Integration
|Level 3

|*Phase 3: Optimierung*
|Quartal 2–4
|Graviton-Migration EC2, Autoscaling + Scale-to-Zero, Batch Time-Shifting, SCI-Messung Tier-1
|Level 3 → 4

|*Phase 4: Continuous Improvement*
|Dauerhaft
|SCI-OKRs, automatisierte Carbon-Pipeline, SBTi-Ziele, jährliche Fortschrittsberichte
|Level 4 → 5

|===

NOTE: Die meisten Organisationen befinden sich beim Start auf Level 1 oder 2. Level 3 ist das *CSRD-Minimum* für berichtspflichtige Unternehmen. Level 4 ist das angestrebte Reifeniveau für technologiegetriebene Unternehmen mit Sustainability-Commitment. Level 5 ist für die meisten Organisationen ein mittelfristiges Ziel (3–5 Jahre).
