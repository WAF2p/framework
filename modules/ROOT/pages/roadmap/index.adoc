= Roadmap
:description: WAF++ version 1.0 and beyond

== Short-term (Phase 1)

* Document the basic reference model
* First version of the maturity model
* Draft a questionnaire catalog for architects
* Formalize governance structure (TSC, maintainers, WGs)

== Medium-term (Phase 2 – Towards v1.0)

* *Maturity Model v1.0* with clear criteria and scoring schema
* *Questionnaire paths:*
** Path for architects
** Path for platform teams
** Path for security/compliance
* *Example implementations* (reference implementations) for typical scenarios
* Definition of an *exam/certification plan* for v1.0

== Exams Roadmap (V1.0 – Concept)

*Foundation-Level Exam:*::
* Basic understanding of WAF++ and the reference model
* Core concepts of the maturity model and governance

*Architect-Level Exam:*::
* Deep dive into architectural decisions
* Applying questionnaires and scoring models to case studies

*Assessor/Auditor Track (optional):*::
* Focus on assessing existing platforms
* Interpreting maturity scores and deriving action plans

== Long-term (Phase 3)

* Integration with other open-source projects (CNCF ecosystem)
* Tools for semi-automated assessment (e.g. questionnaire engines)
* Translations and adaptations for different regulatory contexts
